1. The Structure of a Best-in-Class Prompt
A high-performing prompt should follow this structure to ensure no component is missed: 
Role/Persona: Define the AI as a senior expert (e.g., "You are a world-class Web Strategist and UI/UX Designer").
Goal: Explicitly state the outcome (e.g., "Design a high-converting landing page for [Product]").
Context: Detail the industry, target audience, and unique selling proposition (USP).
Constraints: Set boundaries (e.g., "minimalist style," "max 3-second load time," "WCAG 2.1 compliance").
Components List: Explicitly list the website parts to include. 
2. Every Part of a Website to Include in the Prompt 
Header/Navigation: Simple, intuitive navigation, logo placement, and primary CTA button.
Hero Section (Above the Fold): A clear, value-driven headline (what you do + why care), subheadline, hero image/video, and primary CTA.
Social Proof/Trust Signals: Client logos, testimonials, reviews, case studies, or security badges.
Value Proposition & Benefits: Focusing on benefits over features, using bullet points for readability.
Service/Product Overview: Clearly showcased product highlights.
Conversion Points: Prominent, contrasting buttons (CTA) for booking/purchasing, scattered throughout the page.
Lead Capture/Form: A simple form (1–3 fields) or exit-intent popup.
Footer: Essential links (Privacy Policy, Contact, Social Media).
Technical/Mobile Optimization: Responsive design for mobile (over 60% of traffic) and fast loading speed components.

### Reusable Best-in-Class Prompt Template for Comprehensive Website Design/Audit

Below is a highly structured, reusable prompt template designed to generate best-in-class website designs, audits, or optimizations using AI (e.g., tools like ChatGPT, Grok, or similar). This template incorporates all key components of a website, drawing from UX/UI best practices, CRO principles, SEO strategies, technical optimizations, and psychological elements for high conversion. It's built to be modular—simply fill in the brackets with your specific details (e.g., business name, audience) and reuse it across different websites or sections.

The template follows prompt engineering best practices: assigning a persona, defining goals/context/constraints, listing components exhaustively, using delimiters for clarity, and encouraging iterative outputs. It covers **every part of a website** from header to footer, including above-the-fold elements, content sections, technical aspects, and more. For audits, swap "design" language to "analyze and optimize."

---

**Prompt Template:**

You are a world-class Senior Web Strategist, Lead UI/UX Designer, CRO Expert, and SEO Specialist with over 15 years of experience building high-converting websites for [industry/sector, e.g., e-commerce, SaaS, professional services]. You specialize in creating intuitive, mobile-first designs that drive [primary goal, e.g., lead generation, sales conversions, user engagement] while adhering to best practices like WCAG 2.1 accessibility, Google's Core Web Vitals, and psychological principles such as F-pattern reading, Hick's Law for simplifying choices, and social proof to build trust.

**Goal:** [State the primary outcome, e.g., "Design a complete, high-converting website structure for [business name/service] that maximizes user retention and conversions by at least 30% based on industry benchmarks," or "Audit and optimize an existing website at [URL] for improved SEO, UX, and CRO."] Ensure the design is modern, minimalist, and focused on emotional appeal, urgency, and clear value delivery.

**Context:**
- Target Audience: [Detailed description, e.g., "Busy professionals aged 25-45 in urban areas, tech-savvy, seeking efficient solutions for time management; pain points include overwhelm and lack of productivity tools."]
- Product/Service: [Brief overview, e.g., "A SaaS app for task automation with AI features."]
- Unique Selling Proposition (USP)/Key Value Prop: [What sets it apart, e.g., "The only tool that integrates seamlessly with 50+ apps while using AI to predict user needs, saving 10+ hours/week."]
- Desired Tone and Style: [e.g., "Professional yet approachable, authoritative, playful with subtle humor; use action-oriented language."]
- Brand Guidelines: [e.g., "Color palette: Primary blue (#007BFF), secondary green (#28A745); font: Sans-serif like Roboto."]
- Competitor References: [e.g., "Incorporate elements from high-performing sites like [example.com] for clean navigation and [another.com] for interactive hero sections."]
- Current Website (if auditing): [URL and key issues, e.g., "Existing site at example.com has slow load times and poor mobile responsiveness."]

**Constraints and Guidelines:**
- Visual Hierarchy: Follow F or Z-pattern layouts to guide the user's eye from headline to CTA.
- Mobile-First Design: Ensure fully responsive for devices (60%+ traffic is mobile); prioritize touch-friendly elements and fast loading (under 3 seconds).
- Accessibility: Comply with WCAG 2.1 AA (e.g., alt text for images, sufficient color contrast, keyboard navigation).
- SEO Best Practices: Include optimized meta tags, H1-H6 structure, internal linking, schema markup, and keyword integration without stuffing.
- CRO Principles: Use high-contrast CTAs, scarcity/urgency (e.g., "Limited spots"), A/B testable elements, and exit-intent strategies.
- Technical Specs: Optimize for speed (e.g., compressed images, lazy loading), security (HTTPS, privacy policy), and cross-browser compatibility.
- Copywriting: Concise, benefit-focused (not feature-focused), action-oriented (e.g., "Get Started Now" instead of "Submit"); aim for readability scores above 60 on Flesch-Kincaid.
- Length: Keep sections scannable with bullet points, short paragraphs; total page load under 2MB.
- Prohibitions: No clutter, no auto-play videos, avoid dark patterns like fake urgency.

**Required Components and Structure:**
Provide a detailed wireframe description, suggested copy (headlines, subheadlines, body text, button text), image/video placeholders, and rationale for each section. Ensure sections flow logically to build trust, address pain points, highlight benefits, and drive conversions. Cover every part of the website, including:

1. **Header/Navigation Bar:** Fixed or sticky for easy access; include logo (left-aligned, clickable to home), simple menu (3-5 items max, e.g., Home, Services, About, Blog, Contact), search bar if applicable, and primary CTA button (e.g., "Book a Demo" in contrasting color). Rationale: Intuitive navigation reduces bounce rate by 20-30%.

2. **Hero Section (Above the Fold):** Attention-grabbing area visible without scrolling; include H1 headline (benefit-driven, e.g., "Automate Your Workflow and Reclaim Your Time"), H2 subheadline (expands on value), hero image/video (high-quality, relevant placeholder: "Dynamic illustration of user saving time"), social proof teaser (e.g., "Join 10,000+ happy users"), and prominent CTA button. Rationale: Captures 80% of user attention; apply AIDA (Attention, Interest, Desire, Action) model.

3. **Social Proof/Trust Signals Section:** Build credibility with client logos (carousel or grid), testimonials (3-5 with photos, names, star ratings), reviews (integrated from Google/Trustpilot), case studies (teasers with "Read More" links), or badges (e.g., "Secure SSL," "Money-Back Guarantee"). Rationale: Increases conversions by 15-20% via psychological reciprocity and authority.

4. **Problem/Solution Section:** Empathize with audience pain points (bullet list, e.g., "Struggling with manual tasks?"), then present your solution (how it solves them, with visuals). Rationale: Connects emotionally, using contrast to highlight transformation.

5. **Value Proposition & Benefits Section:** 3-4 columns or bullet points focusing on benefits (e.g., "Save Time: AI predicts tasks" instead of "AI Feature"); include icons or images for scannability. Rationale: Shifts focus from features to user outcomes, reducing cognitive load per Hick's Law.

6. **Service/Product Overview Section:** Detailed showcase with tabs, accordions, or grids; include pricing tiers (if applicable), demos/videos, FAQs. Rationale: Educates users mid-funnel, preventing drop-off.

7. **Conversion Points/CTAs:** Scatter throughout (e.g., after each section); use buttons like "Sign Up Free," forms (minimal fields: name, email), chatbots, or popups (exit-intent for non-intrusive offers). Rationale: Multiple touchpoints boost conversions by guiding users.

8. **Lead Capture/Form Section:** Dedicated area or integrated (e.g., sidebar); simple form with incentives (e.g., "Download Free Guide"); include privacy note. Rationale: Captures leads at peak interest.

9. **Content/Blog Section (if applicable):** Teasers for latest posts with SEO-optimized titles, images, and "Read More" CTAs. Rationale: Drives organic traffic and positions as authority.

10. **About Us/Team Section:** Humanize the brand with bios, photos, mission statement. Rationale: Builds relational trust.

11. **Contact Section:** Map, form, phone/email, hours; integrate with tools like Google Maps. Rationale: Facilitates direct engagement.

12. **Footer:** Bottom navigation (repeat key links), copyright, legal (Privacy Policy, Terms), social media icons, newsletter signup. Rationale: Provides closure and additional conversion opportunities.

13. **Technical/Optimization Elements (Global):** Specify responsive breakpoints, load speed optimizations (e.g., CDN, minified CSS/JS), SEO elements (meta title/description, alt tags), analytics integration (e.g., Google Analytics), and A/B testing recommendations. Rationale: Ensures performance and measurability.

**Output Format:**
- Start with an overall site map/wireframe overview (e.g., in markdown or numbered list).
- For each section: Describe layout (e.g., "Full-width with centered content"), suggested copy, placeholders (e.g., "Image: [description, size: 1200x600]"), hex colors, and CRO/SEO rationale.
- End with implementation tips (e.g., "Use tools like Figma for prototyping") and metrics to track (e.g., bounce rate, conversion rate).
- If needed, suggest iterations: "Version 2: Focus more on emotional storytelling in the hero."

###

**Tips for Reuse and Iteration:**
- Customize brackets for each project.
- For specific pages (e.g., just a landing page), trim irrelevant sections.
- Add few-shot examples: "Make it similar to hubspot.com's clean design."
- If output is generic, follow up with: "Refine with more psychological triggers like scarcity in CTAs."
- Test prompts iteratively to refine based on AI responses.

This template ensures comprehensive coverage, making it reusable for any website project while aiming for best-in-class results. If you provide more details (e.g., your business type), I can customize it further!
